🧠 Reflection & Step Questions

Step 1

  • 🔹 What static content is provided in this version?
    A header, a #words text element, and a red square styled inline with HTML.
  • 🔹 How is the background colour applied using inline JavaScript?
    The background color is applied directly using the style attribute in the HTML.

Step 2

  • 🔹 What is the effect of clicking the square in this version?
    Clicking the square changes its background color using an inline onclick event.
  • 🔹 How does this method of handling events compare to later approaches?
    It mixes behavior and structure, which becomes harder to manage. Later steps use JS-only event handling.

Step 3

  • 🔹 How does the squares background colour change with different events?
    The square turns red on click, green on mouseover, and gray on mouseout using inline event attributes.
  • 🔹 Why might it be useful to define separate functions for these actions?
    It keeps code modular and easier to debug or reuse, rather than duplicating logic inline.

Step 4

  • 🔹 What are the advantages of consolidating the event handling logic into one function?
    A single parameterized function reduces repetition and makes the code easier to maintain.
  • 🔹 How does this approach promote code reusability and maintainability?
    You can use the same function for different events by passing different color values as parameters.

Step 5

  • 🔹 What are the benefits of using addEventListener instead of inline HTML attributes?
    It separates structure from behavior, follows modern best practices, and avoids cluttering HTML.
  • 🔹 How does this improve the separation of concerns between HTML and JavaScript?
    HTML handles layout only, while JS manages interactions, improving readability and flexibility.

Step 6

  • 🔹 What is the purpose of the greeting() function in this version?
    To dynamically insert a welcome message into the page when it loads.
  • 🔹 How does dynamically inserting content into the DOM demonstrate separation of concerns?
    It keeps content logic in JS and layout in HTML, reducing the need for hardcoded values.

Step 7

  • 🔹 Why is it beneficial to remove inline event attributes?
    It improves readability, prevents duplicated logic, and adheres to modern coding standards.
  • 🔹 How does this practice align with best practices for clean and maintainable code?
    It helps separate responsibilities and enables better scalability of the codebase.

Step 8

  • 🔹 How does the external script.js file improve structure?
    It centralizes all logic, keeps HTML clean, and makes it easier to debug and reuse scripts.
  • 🔹 What role does the buzzword generator play?
    It adds a dynamic element and simulates real-time user interaction, enhancing interactivity.
  • 🔹 How does this demonstrate a well-structured, dynamic web app?
    By integrating dynamic content, external logic, and interactive UI, it reflects a complete front-end development workflow.
